# Κράτος.ΝΕΤ

Ανεξάρτητη PWA εφαρμογή για κρατικές υπηρεσίες και ΕΣΠΑ/ΚΑΔ matcher.
Δουλεύει σε κινητό και υπολογιστή, εγκαθίσταται σαν app (PWA),
δεν χρειάζεται build step.

**Δεν σχετίζεται με το Ελληνικό Δημόσιο ή οποιονδήποτε κρατικό φορέα.**

## Τι κάνει σήμερα

1. **Υπηρεσίες (tab)** — deep links σε ~20 βασικές κρατικές πύλες
   (TAXISnet, ΕΦΚΑ, ΔΥΠΑ, ΓΕΜΗ, gov.gr Wallet, Διαύγεια, ΚΗΜΔΗΣ κ.λπ.),
   οργανωμένα σε κατηγορίες.
2. **ΕΣΠΑ/ΚΑΔ (tab)** — βάζεις ΚΑΔ (ή λέξη-κλειδί όπως "καφετέρια") και
   περιφέρεια, σου δείχνει ποια προγράμματα από τα δεδομένα ταιριάζουν,
   με βαθμό σχετικότητας.
3. **Προθεσμίες (tab)** — λίστα των πλησιέστερων ημερομηνιών λήξης
   υποβολής από τα ίδια δεδομένα.

## Τι ΔΕΝ κάνει (ακόμα) και γιατί

- **Δεν κάνει login στις κρατικές υπηρεσίες για σένα.** Το TAXISnet/gov.gr/
  ΕΦΚΑ δεν έχουν δημόσιο API για σύνδεση τρίτης εφαρμογής· θα χρειαζόταν
  αποθήκευση κωδικών σου και screen-scraping login, κάτι νομικά γκρίζο και
  εύθραυστο σε κάθε αλλαγή 2FA. Οι κάρτες είναι deep links στην επίσημη σελίδα.
- **Το ΚΑΔ matching είναι ενδεικτικό, όχι επίσημο.** Το espa.gr δεν εκθέτει
  δομημένα τους επιλέξιμους ΚΑΔ ανά πρόσκληση — συνήθως είναι μόνο μέσα στο
  PDF. Το matching συγκρίνει λέξεις-κλειδιά στον τίτλο/περιγραφή/δικαιούχους.
  Πάντα επιβεβαίωση από το PDF πριν την υποβολή.
- **Τα δεδομένα ΕΣΠΑ είναι seed data (10 πραγματικά προγράμματα από espa.gr,
  1/7/2026)** μέχρι να ενεργοποιήσεις το nightly scraper (βλ. παρακάτω) —
  ο scraper δεν μπόρεσε να τρέξει μέσα σε αυτό το sandbox γιατί δεν έχει
  πρόσβαση στο espa.gr· θα τρέχει κανονικά μέσω GitHub Actions.

## Τι διορθώθηκε (2ος γύρος, πριν το deployment)

- **Pagination fix.** Η espa.gr σελίδα προσκλήσεων είναι ASP.NET WebForms
  με pagination μέσω JavaScript `__doPostBack` — τα σκέτα GET requests
  (παλιά έκδοση scraper) έβλεπαν πάντα την ίδια πρώτη σελίδα (10/288).
  Ο scraper ξαναγράφτηκε με **Playwright** (πραγματικό headless browser),
  που επιλέγει "100 αποτελέσματα/σελίδα" και κάνει κλικ διαδοχικά στις
  σελίδες, ώστε να πιάνει και τα 288 προγράμματα.
- **robots.txt.** Δεν μπόρεσα να διαβάσω το ακριβές περιεχόμενο του
  `espa.gr/robots.txt` από το δικό μου sandbox (μπλοκαρισμένο δίκτυο), οπότε
  αντί να το υποθέσω, ο scraper **το ελέγχει μόνος του σε κάθε run** πριν
  αγγίξει οτιδήποτε — αν κάποιο path είναι disallowed, σταματάει καθαρά
  χωρίς να πειράξει τα υπάρχοντα δεδομένα.
- **Ασφαλέστερο pagination targeting.** Το πρώτο draft θα μπορούσε θεωρητικά
  να κάνει κλικ σε λάθος αριθμητικό link (π.χ. ποσό "100.000€"). Τώρα ο
  εντοπισμός του link για την επόμενη σελίδα γίνεται μόνο μέσα στην περιοχή
  γύρω από τη λέξη "Σελίδα", όχι σε όλο το documento.
- **Δεν χάνονται δεδομένα σε αποτυχία.** Αν ο scraper δεν καταφέρει να
  τραβήξει τίποτα σε κάποιο run (π.χ. το site άλλαξε δομή), ΔΕΝ σβήνει το
  υπάρχον `data/programs.json` — κρατάει τα παλιά δεδομένα και βγάζει
  exit code 1 ώστε το GitHub Action να φανεί ως failed (θα το δεις).

### Γνωστός περιορισμός που ΔΕΝ έχει λυθεί ακόμα

Ο εντοπισμός των συνδέσμων σελιδοποίησης (`find_next_page_href`) βασίζεται
σε ευρετική αναζήτηση γύρω από τη λέξη "Σελίδα" επειδή δεν είχα πρόσβαση
στο πραγματικό raw HTML/CSS classes της σελίδας κατά την ανάπτυξη (μόνο σε
extracted-markdown προεπισκόπηση). Είναι πολύ πιθανό να δουλέψει, αλλά αν
στο πρώτο πραγματικό run στο GitHub Actions δεις στα logs κάτι σαν
`[ok] σελίδα 1: σύνολο 10 προγράμματα` και μετά `Τέλος σελιδοποίησης` (δηλ.
σταμάτησε στη σελίδα 1 αντί να συνεχίσει), σημαίνει ότι η ευρετική δεν
"έπιασε" το pager container — φέρε μου τα logs και το διορθώνουμε μαζί σε
2 λεπτά (πιθανότατα θα χρειαστεί μόνο μια μικρή αλλαγή στο selector).

## Νέα χαρακτηριστικά: Dark mode + "Κοντά μου"

### Dark mode
Toggle πάνω δεξιά (🌓). Θυμάται την επιλογή σου (localStorage) και ξεκινάει σύμφωνα με τη
ρύθμιση συστήματος του κινητού/browser αν δεν έχεις επιλέξει κάτι χειροκίνητα.

### "Κοντά μου" tab
- **Πρατήριο καυσίμων / ATM**: πλήρως αυτόματο. Παίρνει το GPS σου, ρωτάει το δημόσιο
  Overpass API (OpenStreetMap) για τα κοντινότερα σημεία σε ακτίνα 6χλμ, υπολογίζει
  πραγματικές αποστάσεις και τα δείχνει ταξινομημένα. Κάθε αποτέλεσμα έχει 3 κουμπιά —
  **Google Maps / Waze / Apple Maps** — ώστε να διαλέξεις εσύ ποια εφαρμογή πλοήγησης θα
  ανοίξει με έτοιμη διαδρομή.
- **Εφημερεύον φαρμακείο / νοσοκομείο**: εδώ ήμουν ειλικρινής σε ένα όριο — δεν υπάρχει
  δημόσιο API για το πρόγραμμα εφημερίας (είναι σύστημα βάρδιας που διαχειρίζεται κάθε
  τοπικός φαρμακευτικός σύλλογος / το Υπουργείο Υγείας ξεχωριστά, όχι μία κεντρική βάση
  δεδομένων). Αντί να το παραλείψω, χρησιμοποιεί το GPS σου + reverse geocoding (Nominatim/
  OpenStreetMap) για να αναγνωρίσει την πόλη σου, και σε πάει κατευθείαν σε
  προ-συμπληρωμένη αναζήτηση Google Maps/Waze/Apple Maps + επίσημες πηγές
  (moh.gov.gr για νοσοκομεία, farmakeia.gr/vrisko.gr για φαρμακεία). Ένα tap λιγότερο
  από το να το ψάχνεις μόνος σου, όχι ψευδής αυτοματοποίηση κάτι που δεν υπάρχει.
- Καμία τοποθεσία δεν αποθηκεύεται πουθενά — ζει μόνο στη μνήμη της σελίδας όσο είναι ανοιχτή.

### Εξαρτήσεις που προστέθηκαν
- **Overpass API** (overpass-api.de) — δωρεάν, χωρίς API key, δημόσιο OSM endpoint.
- **Nominatim** (nominatim.openstreetmap.org) — δωρεάν reverse geocoding. Έχει fair-use
  policy (μέχρι ~1 request/δευτερόλεπτο)· για προσωπική χρήση είναι μια χαρά, αλλά αν το
  app αποκτήσει πολλούς χρήστες ταυτόχρονα καλό είναι να μεταβούμε σε δικό μας
  Nominatim instance ή πληρωμένο geocoding provider (π.χ. Google/Mapbox).
- Οι κλήσεις αυτές γίνονται απευθείας από τον browser του κάθε χρήστη — δεν περνάνε από
  δικό μας server, άρα δεν χρειάζεται κανένα νέο secret/backend.

## Γύρος 4: Offline, Διαύγεια, "ΝΕΟ" badges, Ημερολόγιο

### 1. Πραγματικό offline (Service Worker)
Το `sw.js` κάνει cache το app shell (HTML/CSS/fonts/icons) και τα δεδομένα ΕΣΠΑ με
stale-while-revalidate (δείχνει τα cached δεδομένα αμέσως, τα ανανεώνει στο παρασκήνιο).
Λειτουργεί μόνο πάνω σε HTTPS (Cloudflare Pages το καλύπτει αυτόματα) — σε `file://` ή
`http://` χωρίς SSL δεν ενεργοποιείται.

### 2. Shortcuts στο εικονίδιο εφαρμογής
Long-press (κινητό) ή δεξί-κλικ (desktop) στο εικονίδιο kratosNet μετά την εγκατάσταση →
3 συντομεύσεις: Εφημερεύον φαρμακείο, Κοντινό ATM, ΕΣΠΑ/ΚΑΔ. Δουλεύουν μέσω URL hash
(`#nearby=pharmacy` κ.λπ.) — αν το δικό σου OS/browser δεν υποστηρίζει ακόμα PWA shortcuts,
απλά δεν θα εμφανιστούν, χωρίς να σπάει τίποτα.

### 3. Δήμος Αγρινίου — ζωντανή ροή Διαύγειας (tab "Δήμος")
Χρησιμοποιεί το επίσημο API της Διαύγειας (`organizationUid:"6012"` = Δήμος Αγρινίου,
επιβεβαιωμένο). **Ειλικρινές disclaimer**: το ακριβές σχήμα του JSON response δεν μπόρεσα
να το επιβεβαιώσω πλήρως κατά την ανάπτυξη (περιορισμοί δικτύου στο dev sandbox — μόνο το
`wt=xml` export URL μπόρεσα να δοκιμάσω, όχι το `wt=json`). Ο κώδικας κάνει defensive
parsing (δοκιμάζει πολλαπλά πιθανά ονόματα πεδίων) και αν αποτύχει, δείχνει καθαρά «δεν
φόρτωσε» + κουμπί απευθείας στο diavgeia.gov.gr — ποτέ δεν κρασάρει, ποτέ δεν δείχνει κάτι
λάθος σαν να είναι σωστό. Αν στο πρώτο πραγματικό deployment δεις σταθερά το fallback
μήνυμα αντί για πραγματικά αποτελέσματα, στείλε μου τι δείχνει το browser console
(Network tab → το αίτημα προς diavgeia.gov.gr) και το προσαρμόζουμε άμεσα.

### 4. "ΝΕΟ" badges + απομνημόνευση ΚΑΔ/περιφέρειας + εξαγωγή σε ημερολόγιο
- Κάθε φορά που ανοίγεις το ΕΣΠΑ tab, το app συγκρίνει τα τρέχοντα προγράμματα με όσα
  είχες ξαναδεί (localStorage) και βάζει ετικέτα **ΝΕΟ** σε ό,τι εμφανίστηκε από τότε.
  Στην πρώτη επίσκεψη δεν βάζει ΝΕΟ σε τίποτα (δεν έχει τι να συγκρίνει).
- Ο ΚΑΔ και η περιφέρεια που βάζεις θυμούνται αυτόματα (localStorage) και προσυμπληρώνονται
  + τρέχει αυτόματα η αναζήτηση στην επόμενη επίσκεψη — δοκιμασμένο, δουλεύει.
- Κάθε αποτέλεσμα με γνωστή ημερομηνία λήξης έχει κουμπί **"📅 Στο ημερολόγιο"** που
  κατεβάζει ένα `.ics` αρχείο (συμβατό με Google/Apple/Outlook Calendar).
- Δεν πρόσθεσα push notifications: στο iOS Safari οι PWA δεν έχουν αξιόπιστο background
  push χωρίς Apple Developer account + APNs setup (μεγάλο side-project). Το `.ics` export
  είναι πιο αξιόπιστη λύση σήμερα — το ημερολόγιο του κινητού σου θα σε ειδοποιήσει κανονικά.

## Γύρος 5: Μία έκδοση, ρυθμιζόμενος δήμος

Αντί για δύο ξεχωριστά builds (Αγρίνιο vs "όποιος δήμος"), το tab **"Δήμος"** είναι τώρα
ρυθμιζόμενο:
- **Εξ ορισμού** δείχνει Δήμο Αγρινίου (organizationUid 6012) — δουλεύει έτοιμο για σένα,
  χωρίς καμία ρύθμιση.
- Κάθε πολίτης που το εγκαθιστά μπορεί να πατήσει το ⚙︎ δίπλα στον τίτλο του tab και να
  βάλει το δικό του δήμο (όνομα + το `organizationUid` του από το diavgeia.gov.gr).
  Αποθηκεύεται τοπικά στη συσκευή του (localStorage) — δεν επηρεάζει κανέναν άλλον χρήστη.
- **Γιατί όχι πλήρης αυτόματη λίστα των 332 δήμων**: το Διαύγεια δεν έχει δημόσια
  αναζήτηση-με-όνομα που να μπορώ να επιβεβαιώσω αξιόπιστα, και η πλήρης λίστα φορέων
  περιλαμβάνει χιλιάδες εγγραφές (όχι μόνο δήμους) χωρίς σαφές φίλτρο τύπου. Θα ήταν
  ανέντιμο να παραδώσω μια "λίστα δήμων" που δεν έχω πραγματικά επαληθεύσει. Αν θες, σε
  επόμενο γύρο μπορούμε να χτίσουμε σιγά-σιγά μια επαληθευμένη λίστα των πιο κοντινών/
  συχνά ζητούμενων δήμων (π.χ. όλης της Δυτικής Ελλάδας), ή να δούμε αν υπάρχει τρόπος
  φιλτραρίσματος στο Διαύγεια που να τον έχω παραβλέψει.
- Αν θες αυστηρά **ξεχωριστό, "κλειδωμένο" deployment** μόνο για τον Δήμο Αγρινίου (π.χ.
  για επίσημη χρήση/branding χωρίς να φαίνεται το κουμπί ρυθμίσεων σε πολίτες άλλων δήμων),
  είναι μια μικρή αλλαγή — κρύβουμε το ⚙︎ με ένα build flag. Πες μου αν το θες και το
  προσθέτω.

## Γύρος 6: Πλοήγηση εντός εφαρμογής (in-app viewer)

Οι κάρτες δεν ανοίγουν πια όλες σε νέα καρτέλα browser — πολλές ανοίγουν **μέσα στο
kratosNet**, σε ένα full-screen modal με κουμπί κλεισίματος (✕) και εναλλακτικό
"Άνοιγμα σε browser" αν κάτι δεν φορτώσει σωστά.

**Ανοίγουν εντός εφαρμογής** (ενημερωτικές σελίδες, καμία εισαγωγή κωδικών):
gov.gr αρχική, ΚΕΠ, ΓΕΜΗ, Έλεγχος ΚΑΔ, ependyseis.gr, espa.gr προσκλήσεις (+ κάθε
μεμονωμένη πρόσκληση από το ΕΣΠΑ tab), Solon.gr, Διαύγεια (+ κάθε έγγραφο από το tab
"Δήμος"), ΚΗΜΔΗΣ, Δήμος Αγρινίου, e-Συνταγογράφηση (ενημερωτικό info-page, όχι το ίδιο
το σύστημα συνταγογράφησης).

**Παραμένουν σε πλήρη browser** (login, κωδικοί, πληρωμές): TAXISnet, myAADE, myDATA,
e-Παράβολο, gov.gr Wallet, gov.gr Sign, ΕΦΚΑ, ΔΥΠΑ, ΕΡΓΑΝΗ, e-Ραντεβού, MyAuto, ΚΤΕΟ.

**Γιατί αυτός ο διαχωρισμός, και όχι "όλα εντός εφαρμογής"**: έλεγξα τεχνικά — καμία
από τις κρατικές σελίδες δεν στέλνει header που να μπλοκάρει το iframe (θα μπορούσα να
τα βάλω όλα εντός εφαρμογής χωρίς να με σταματήσει τίποτα τεχνικά). Αλλά το να βάζεις
τους κωδικούς TAXISnet ή να κάνεις πληρωμή μέσα σε ένα iframe τυλιγμένο σε δικό μου
branding είναι ακριβώς το pattern που χρησιμοποιείται σε phishing επιθέσεις (clickjacking) —
ο χρήστης δεν μπορεί να επαληθεύσει εύκολα ότι βρίσκεται πραγματικά στην επίσημη σελίδα.
Προτίμησα την ασφαλή επιλογή αντί για το "όλα in-app επειδή μπορώ". Επιπλέον, πολλά login
συστήματα σπάνε μέσα σε iframe λόγω σύγχρονων περιορισμών cross-site cookies των browsers
(SameSite policy) — άρα ακόμα κι αν το επιχειρούσαμε, το login πιθανόν να μην δούλευε αξιόπιστα.

Τα κουμπιά πλοήγησης (Google Maps/Waze/Apple Maps) παραμένουν εξωτερικές εφαρμογές —
αυτό δεν αλλάζει, δεν υπάρχει τρόπος να "μπει" το Google Maps μέσα σε μια web εφαρμογή,
είναι διαφορετικό, ξεχωριστό app.

**Προσθήκη διαφάνειας**: το viewer bar δείχνει πάντα το πραγματικό domain (π.χ. 🔒
`www.gov.gr`) δίπλα στον φιλικό τίτλο, ώστε να είναι ξεκάθαρο πού βρίσκεσαι ακόμα και
εντός εφαρμογής — μικρή αλλά σημαντική λεπτομέρεια διαφάνειας.

## Γύρος 7: Fix για crash σε sandboxed preview περιβάλλοντα

Βρέθηκε πραγματικό bug: σε αυστηρά sandboxed iframes (όπως πολλά preview panels,
χωρίς το `allow-same-origin` flag), το `localStorage` πετάει σφάλμα **στην ίδια την
πρόσβαση** αντί απλά να μη δουλεύει — και το app το καλούσε χωρίς προστασία σε 12
σημεία, με το πρώτο μάλιστα να τρέχει στην πρώτη γραμμή του script (theme init).
Πρόσθεσα ένα `safeStorage` wrapper (try/catch παντού) και ένα αντίστοιχο guard στο
service worker registration. Δοκιμάστηκε εκτενώς: πλήρες interactive flow (tabs,
ρυθμίσεις δήμου, viewer, theme toggle) μέσα σε προσομοιωμένο μέγιστα περιοριστικό
sandboxed iframe — μηδέν σφάλματα τώρα. Αν ξαναδείς πρόβλημα, πες μου ακριβώς πού
το βλέπεις (ποιο browser/συσκευή, ποια ενέργεια το προκαλεί, τι μήνυμα δείχνει) ώστε
να το εντοπίσω γρήγορα.

## Γύρος 8: Λογαριασμοί, συγχρονισμός, email ειδοποιήσεις ΚΑΔ — όλα σε δωρεάν tiers

### Τι προστέθηκε
1. **Σύνδεση με email (magic link)** — χωρίς κωδικούς, στο ΕΣΠΑ tab κάτω από τα
   αποτελέσματα. Ο ΚΑΔ, η περιφέρεια, ο δήμος και η ρύθμιση ειδοποιήσεων
   συγχρονίζονται αυτόματα σε όλες τις συσκευές του χρήστη.
2. **Νυχτερινές email ειδοποιήσεις**: το υπάρχον GitHub Actions workflow, μετά το
   scrape, τρέχει το `scraper/notify_kad.py` — βρίσκει νέα προγράμματα που ταιριάζουν
   στον ΚΑΔ κάθε opt-in χρήστη και στέλνει ΕΝΑ συγκεντρωτικό email (όχι spam ανά
   πρόγραμμα). Κρατάει ιστορικό ώστε να μην ξαναστείλει ποτέ το ίδιο.
3. Αν δεν έχει γίνει ακόμα το setup, το app δείχνει ήσυχα "δεν έχει ενεργοποιηθεί" —
   τίποτα δεν σπάει.

### Κόστος: 0€ — και τι σημαίνει αυτό ακριβώς
| Υπηρεσία | Free tier | Όριο πρακτικά | Τι συμβαίνει αν ξεπεραστεί |
|---|---|---|---|
| Supabase | 50k MAU, 500MB DB | χιλιάδες χρήστες | Παύει να δέχεται νέα — ΔΕΝ χρεώνει |
| Brevo | 300 emails/ημέρα | ~300 opt-in χρήστες | Τα επιπλέον πάνε αύριο — ΔΕΝ χρεώνει |
| GitHub Actions | 2000 min/μήνα | το nightly τρέχει ~5min = 150min/μήνα | — |
| Cloudflare Pages | unlimited static | — | — |

**Καμία από αυτές δεν έχει αυτόματη μετάβαση σε πληρωμή** — όλες σταματούν αντί να
χρεώνουν. Μοναδική προσοχή: το Supabase free κάνει **pause το project μετά από 7 ημέρες
πλήρους αδράνειας** — το nightly notification script το κρατάει ενεργό από μόνο του.

### Setup (μία φορά, ~15 λεπτά)

**Α. Supabase** (supabase.com → New project → Free tier)
1. SQL Editor → New query → επικόλλησε ΟΛΟ το `supabase/schema.sql` → Run
2. Authentication → Providers → Email: βεβαιώσου ότι είναι ενεργό (είναι by default).
   *Σημείωση: τα auth emails του free tier έχουν όριο ~4/ώρα ανά project — αρκετό για
   αρχή· αν γίνει πρόβλημα, συνδέεις το Brevo SMTP και στο Supabase auth (δωρεάν κι αυτό).*
3. Settings → API: αντέγραψε **Project URL** και **anon public key**
4. Άνοιξε το `index.html`, βρες το `SUPABASE_CONFIG` (κοντά στην αρχή του Supabase
   section στο script) και συμπλήρωσε τα δύο πεδία. Commit + push. (Το anon key είναι
   σχεδιασμένο να είναι δημόσιο — η προστασία γίνεται από τα Row Level Security
   policies στη βάση.)
5. Authentication → URL Configuration → Site URL: βάλε το pages.dev URL σου
   (για να δουλεύουν σωστά τα magic links)

**Β. Brevo** (brevo.com → Sign up free, χωρίς κάρτα)
1. Επιβεβαίωσε ένα sender email (Settings → Senders — μπορεί να είναι το Gmail σου)
2. SMTP & API → API Keys → Generate new key

**Γ. GitHub Secrets** (repo → Settings → Secrets and variables → Actions)
| Secret | Τιμή |
|---|---|
| `SUPABASE_URL` | το Project URL |
| `SUPABASE_SERVICE_KEY` | Settings → API → **service_role** key (ΠΟΤΕ στο frontend!) |
| `BREVO_API_KEY` | από το βήμα Β2 |
| `NOTIFY_SENDER_EMAIL` | το verified sender email |
| `APP_URL` | το pages.dev URL |

Αν κάποιο secret λείπει, το notification βήμα απλά κάνει skip με καθαρό μήνυμα στα
logs — δεν αποτυγχάνει το workflow.

**Σημείωση δοκιμών**: η λογική matching του notifier είναι unit-tested με mock δεδομένα
(4/4 σενάρια σωστά, συμπεριλαμβανομένου του φιλτραρίσματος περιφέρειας). Η πλήρης
end-to-end ροή (πραγματικό Supabase → πραγματικό email) δεν μπορούσε να δοκιμαστεί
από το dev περιβάλλον — στο πρώτο πραγματικό run, τρέξε το workflow χειροκίνητα
(Actions → Run workflow) και δες τα logs· αν κάτι θέλει προσαρμογή, είναι πιθανότατα
1-2 γραμμές.

### Payment layer (βήμα 3 — ΔΕΝ έγινε ακόμα, συνειδητά)
Πρώτα χρειάζεται το παραπάνω να τρέξει και να αποκτήσει χρήστες. Όταν είσαι έτοιμος:
το καθαρότερο δωρεάν-για-αρχή μονοπάτι είναι **Stripe Payment Links** (καμία μηνιαία
χρέωση, μόνο προμήθεια ανά συναλλαγή) στη **web** έκδοση, με ένα `is_premium` πεδίο
στο profiles και έλεγχο στο notifier (π.χ. free = 1 ΚΑΔ, premium = πολλαπλοί ΚΑΔ για
λογιστές). Μέσα στο Play Store app θα χρειαστεί Google Play Billing — το αφήνουμε
για όταν υπάρχει πραγματική ζήτηση.

## Προετοιμασία για Google Play Store

Η πιο γρήγορη διαδρομή για μια PWA σαν το kratosNet είναι το **PWABuilder** — δεν
χρειάζεται δικό σου Android SDK/toolchain, όλα γίνονται online. Χρειάζεται **πρώτα να
έχεις κάνει το βήμα 1-2 deployment** παραπάνω (χρειάζεται πραγματικό HTTPS URL).

### Βήμα 1: Δημιουργία του Android πακέτου
1. Πήγαινε στο [pwabuilder.com](https://www.pwabuilder.com)
2. Βάλε το `pages.dev` (ή custom domain) URL σου
3. Θα σου δείξει audit του manifest — το δικό μας manifest.json είναι ήδη έτοιμο
   (icons 192/512 + maskable variant για Android adaptive icons, ήδη μέσα στο repo)
4. **Package for stores → Android**
5. Θα σου ζητήσει: Package ID (πρότεινε κάτι σαν `gr.kratosnet.twa`), signing key
   (μπορεί να σου φτιάξει νέο ή να ανεβάσεις δικό σου .keystore)
6. Κατεβάζεις ένα **.aab** (Android App Bundle) — αυτό ανεβαίνει στο Play Console

### Βήμα 2: Επαλήθευση domain (Digital Asset Links)
Για να ανοίγει η εφαρμογή **χωρίς** γραμμή διευθύνσεων browser (πλήρες native app feel),
το Android χρειάζεται επιβεβαίωση ότι εσύ ελέγχεις και το domain και την εφαρμογή:
1. Το PWABuilder θα σου δώσει ένα **SHA256 fingerprint** μετά τη δημιουργία του πακέτου
2. Άνοιξε `.well-known/assetlinks.json` (ήδη υπάρχει στο repo, template) και αντικατέστησε
   το `REPLACE_WITH_SHA256_FINGERPRINT_...` με το πραγματικό fingerprint
3. Κάνε commit + push — θα είναι προσβάσιμο στο
   `https://kratosnet-xxx.pages.dev/.well-known/assetlinks.json` αυτόματα

### Βήμα 3: Google Play Console (δικά σου βήματα, δεν μπορώ να τα κάνω εγώ)
1. Λογαριασμός [Google Play Console](https://play.google.com/console) — εφάπαξ κόστος $25
2. **Create app** → ανέβασε το `.aab` από το Βήμα 1
3. Χρειάζεσαι: εικονίδιο 512×512 (έχουμε το `icon-512.png`), screenshots από το app
   (τραβάς εσύ μετά την εγκατάσταση), σύντομη + πλήρη περιγραφή, **Privacy Policy URL**
   (υποχρεωτικό — χρειάζεται μια απλή σελίδα, μπορώ να σου γράψω το κείμενο αν θες),
   κατηγορία (Government/Productivity), content rating ερωτηματολόγιο
4. Review από Google: συνήθως 1-7 ημέρες για πρώτη υποβολή

### Εναλλακτική: Capacitor (αν θες πιο βαθύ έλεγχο)
Αφού ήδη χρησιμοποιείς Capacitor στο Lemon Juice/KOFI POS, θα μπορούσαμε εναλλακτικά
να το πακετάρουμε έτσι αντί για PWABuilder — δίνει πιο άμεση πρόσβαση σε native APIs
(π.χ. πιο αξιόπιστο geolocation, background tasks) αλλά χρειάζεται τοπικό Android
Studio/SDK setup στον υπολογιστή σου για να χτίσεις το τελικό .aab. Πες μου αν το
προτιμάς αυτό και σου ετοιμάζω το `capacitor.config.ts` + οδηγίες.
